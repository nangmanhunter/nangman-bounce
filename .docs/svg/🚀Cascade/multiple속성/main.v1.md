크롬에서 갑자기 여러 개가 다 잡히는 핵심 이유는 바로 `<input>` 태그에 추가된 **`multiple` 이라는 단 한 단어 속성** 때문입니다!

아주 단순해 보이지만, 브라우저 환경에서는 이 속성 하나가 완전히 다른 세상을 만듭니다. 아까 코드와 지금 코드의 차이를 딱 직관적으로 보여드릴게요.

---

### 1. 태그의 차이 (브라우저에게 내리는 명령)

* **아까 코드:** `<input type="file" ... >`
* 브라우저는 이걸 보고 "아, 사용자가 무조건 파일 '한 개'만 올릴 건가 보네"라고 판단합니다. 그래서 파일 선택창이 떴을 때 `Ctrl`이나 `Shift`를 누르고 아무리 다른 파일을 클릭해도 선택이 안 되고 취소됐던 겁니다.


* **지금 코드:** `<input type="file" multiple ... >`
* 뒤에 `multiple`이 붙는 순간, 크롬은 **"오, 이 사이트는 한 번에 여러 개 올려도 되는 곳이구나!"** 하고 알아먹습니다. 그래서 드디어 크롬 자체에서 파일 여러 개를 파랗게 다 잡아버릴 수 있도록 허락해 주는 것입니다.



---

### 2. 스크립트의 차이 (자바스크립트가 일하는 방식)

크롬이 3개의 파일을 잡아다 준 뒤, 자바스크립트가 처리하는 방식도 완전히 달라졌습니다.

* **아까 코드 (`files[0]`)**
```javascript
const file = event.target.files[0]

```


* 만약에 아까 코드에 `multiple`이 있어서 크롬이 파일 3개를 가져다줬더라도, 코드에서 대놓고 "난 맨 앞에 있는 0번째 녀석만 일 시킬 거야"라고 선언해 버렸기 때문에 결국 다운로드는 1개밖에 안 됐을 겁니다.


* **지금 코드 (`Array.from`)**
```javascript
Array.from(files).forEach(file => { ... })

```


* 지금 코드는 크롬이 "자, 여기 파일 3개 잡아왔어!" 하고 보따리를 넘겨주면, "오케이, 3개 다 가져와서 전부 다 동시에 변환해!"라고 명령을 내리는 구조입니다.
