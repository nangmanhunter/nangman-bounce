아, 어떤 개념으로 나누려고 하시는지 단번에 이해했습니다!

개발이나 프로젝트에서 **정형화된 딱딱한 모양**과 **자유롭고 복잡한 곡선형 모양**을 직관적으로 대비시키고 싶으신 거군요.

그렇다면 **Shape**와 **Character**의 조합보다는, 영문 개발 명명법(Naming)이나 UI/그래픽 데이터 구조에서 이 둘을 완벽하게 대비해 주는 **더 직관적인 단어 쌍**들이 있습니다. 구현하려는 느낌에 맞춰 골라보세요.

---

### 1. [가장 추천] 딱딱한 도형 vs 복잡한 유선형/패스

데이터나 모듈을 분기할 때 가장 직관적이고 직업적인(Professional) 표현입니다.

* **Shape vs Path (패스)**
* **Shape**: 원, 네모, 세모처럼 수식(반지름, 가로, 세로)으로 딱 떨어지는 **'기하학적 도형'**
* **Path**: 펜툴로 딴 것처럼 복잡한 곡선, 유선형, 자유로운 형태의 **'벡터 경로'**
* *추천 이유:* 일러스트레이터나 SVG, Canvas 가이드라인에서 '도형'과 '복잡한 유선형'을 나눌 때 공식적으로 쓰는 표준적인 상대어입니다.


* **Shape vs Mesh (메쉬) 또는 Contour (콘투어)**
* **Shape**: 단순 평면 도형
* **Mesh/Contour**: 굴곡이 있고 복잡한 유선형의 형상/외곽선



---

### 2. Character를 꼭 쓰고 싶다면: Character vs Geometry

만약 `Character`라는 단어를 이미 코드나 기획에서 핵심 키워드로 잡으셨다면, '네모 세모'를 뜻하는 상대어는 **Geometry**나 **Primitive**가 좋습니다.

* **Geometry (기하학/도형) vs Character (특성/유선형)**
* 수학과 물리학을 하셨으니 이 뉘앙스가 가장 와닿으실 겁니다. `Geometry`는 딱딱하게 계산되는 규칙적인 도형(Square, Triangle 등)의 집합을 뜻하고, 이에 대비되는 복잡하고 고유한 유선형의 형태를 `Character(고유한 특징을 가진 형상)`로 두는 방식입니다.


* **Primitive (기본 도형) vs Character (복잡한 형상)**
* 그래픽스에서 네모, 세모, 구 같은 기본 도형을 `Primitive`라고 부릅니다.

### 💡 요약 및 추천

* **코드 데이터 구조나 타입을 나눌 때:** `Shape` (네모/세모)  vs  `Path` (복잡한 유선형)
* **개념적인 오브젝트 분류일 때:** `Geometry` (네모/세모)  vs  `Character` (복잡한 유선형)

만약 지금 만드시는 모듈의 UI나 데이터 타입을 정의하시는 거라면 **Shape vs Path** 조합이 제3자가 코드를 봤을 때도 "아, 이건 기본 도형이고 이건 복잡한 곡선이구나" 하고 가장 오해 없이 한눈에 이해할 수 있는 깔끔한 네이밍이 될 것 같습니다!
